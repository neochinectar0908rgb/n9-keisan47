# 数式組合せ検索ツール

入力した整数（0〜999）に対して、条件を満たす数字の組をすべて探して表示するスマホ向け Web アプリ（PWA）です。
画面上部のボタンで **3つのモード** を切り替えて使えます。

### ① 通常モード（×××＋／−）

- **式1**　a × b × c ＋ d ＝ 入力値
- **式2**　j × k × l − m ＝ 入力値

足し算型・引き算型の結果を上下 **2つの欄** に分けて表示します。
d・m は 1〜9（0は使いません）。

### ② 積のみモード（×××）

- **式**　a × b × c ＝ 入力値

3つの異なる数字の積だけを検索し、結果欄は **1つだけ** 表示します（`+0`・`-0` は表示しません）。

### ③ 2数＋／−モード（××＋／−）

- **式1**　a × b ＋ d ＝ 入力値
- **式2**　j × k － m ＝ 入力値

掛け算を **2つの数** だけで行うモードです（3つ目の掛ける数 c・l は使いません）。
足し算型・引き算型の結果を **2つの欄** に分けて表示します。
a, b, d（および j, k, m）は互いに異なり、d・m は 1〜9（0は使いません）。a < b、j < k の昇順で表示します。

### 共通の決まり

各数字は 1〜9 の整数。1つの式の中では同じ数字を2回以上使いません。
掛け算部分の順番違いは同じ組として扱い、昇順で1件だけ表示します（例「2×4+3」と「4×2+3」は同じ1件）。
初期状態は通常モードです。モードを切り替えると古い検索結果は消え、入力値はそのまま残ります。

## 💻 ローカル（自分のパソコン）での確認方法

### 方法A：とりあえず計算だけ試す（かんたん）

`index.html` をダブルクリックしてブラウザで開くだけ。
数字を入れて検索できます。
※ ただし `file://` で開くため、**PWA（オフライン・ホーム画面追加）の機能は動きません**。

### 方法B：PWA（オフライン等）まで含めて確認する（おすすめ）

PWA はセキュリティ上 `http://` または `https://` でないと動きません。
付属の `server.ps1` で簡易サーバーを立てると確認できます。

1. このフォルダを開く
2. アドレスバーに `powershell` と入力して PowerShell を起動（またはスタートメニューから）
3. 次のコマンドを実行（フォルダの場所は自分の環境に合わせてください）

```powershell
powershell -ExecutionPolicy Bypass -File "server.ps1"
```

4. ブラウザで `http://localhost:8123/` を開く
5. 止めるときは PowerShell の画面で `Ctrl + C`

### ロジックの自動テストを実行する

```powershell
powershell -ExecutionPolicy Bypass -File "test.ps1"
```

入力値 0〜999 のすべて（通常モード1000＋積のみモード1000＋2数＋／−モード1000＝**3000ケース**）を、
アプリと同じ方式（昇順の組合せループ）と、**わざと別方式**（全順列→ソート→重複排除）の参照ロジックで
二重に計算し、完全一致するかを比較します。
`==> すべて成功` と表示されれば、3モードの検索ロジックは正常です。

## 🌐 GitHub Pages で公開する手順（プログラミング初心者向け）

「GitHub Pages」は、無料でホームページ（Web アプリ）を公開できるサービスです。
スマホからアクセスできる URL がもらえます。

### ステップ1：GitHub アカウントを作る

1. スマホかパソコンで [https://github.com](https://github.com) を開く
2. 右上の **「Sign up」** をタップ
3. メールアドレス・パスワード・ユーザー名を入力して登録
4. 届いたメールの確認を済ませる

### ステップ2：リポジトリ（保存場所）を作る

「リポジトリ」とは、ファイルを置く入れ物のことです。

1. ログイン後、画面右上の **「＋」→「New repository」** をタップ
2. **Repository name（リポジトリ名）** を入力（例：`keisan-tool`）
   - 半角英数字とハイフンがおすすめ
3. **Public（公開）** を選ぶ ← GitHub Pages には Public が必要
4. 「Add a README file」などは **チェックしないまま**でOK
5. 緑色の **「Create repository」** をタップ

### ステップ3：作ったファイルをアップロードする

1. 作成したリポジトリの画面で **「Add file」→「Upload files」** をタップ
2. 公開に必要な次の6ファイルを、まとめてドラッグ＆ドロップ（またはファイル選択）
   - `index.html`
   - `manifest.json`
   - `service-worker.js`
   - `icon-192.png`
   - `icon-512.png`
   - `icon-maskable-512.png`
   - （`README.md` も入れておくと説明が表示されて便利）
3. 画面下の **「Commit changes」**（緑のボタン）をタップして確定

> 💡 `test.ps1` と `server.ps1` はパソコン確認用なので、アップロードしてもしなくても構いません。

### ステップ4：GitHub Pages を有効にする

1. リポジトリの上部メニューから **「Settings」** をタップ
2. 左メニュー（または下の方）の **「Pages」** をタップ
3. **「Build and deployment」** の **Source** を **「Deploy from a branch」** に設定
4. **Branch** を **「main」**、フォルダを **「/ (root)」** に設定して **「Save」**
5. 少し待つ（1〜数分）と、ページ上部に公開 URL が表示されます
   - 形式：`https://（ユーザー名）.github.io/（リポジトリ名）/`
   - 例：`https://taro123.github.io/keisan-tool/`

> このアプリは**相対パス**で作ってあるので、サブディレクトリ（リポジトリ名つきの URL）でも正しく動きます。

### ステップ5：スマホで開く

1. スマホの **Chrome** を開く
2. ステップ4で表示された公開 URL を入力して開く
   - URL が長いときは、パソコンから自分宛にメール・LINE 等で送ると楽です

### ステップ6：Android Chrome でホーム画面に追加（アプリ化）

1. 公開 URL を Chrome で開いた状態で、右上の **「⋮（メニュー）」** をタップ
2. **「ホーム画面に追加」** または **「アプリをインストール」** をタップ
3. 名前を確認して **「追加」** をタップ
4. ホーム画面にアイコンができ、タップするとアプリのように全画面で起動します
   - アドレスバーが目立たない表示になります（standalone 表示）

### ステップ7（必要なとき）：更新を反映する

ファイルを修正して公開内容を新しくしたいときは：

1. GitHub のリポジトリで **「Add file」→「Upload files」** から、修正したファイルを同じ名前でアップロード
   - 同名ファイルは上書きされます
2. **「Commit changes」** で確定 → 数分で公開ページに反映されます

> ⚠️ **キャッシュについての大事な注意**
> このアプリはオフライン対応のため、ファイルをスマホ側にキャッシュ（保存）しています。
> `index.html` などの中身を変更したときは、**`service-worker.js` の中の**
> `var CACHE_NAME = "suushiki-tool-v4";`
> の **`v4` を `v5`, `v6`… と必ず増やしてからアップロード**してください。
> こうすると古いキャッシュが自動で消え、新しい内容が確実に反映されます。
> （番号を変えないと、古い画面が残り続けることがあります）

## 🔢 計算の仕組み（参考）

- 掛け算用の数字は「組合せ」として昇順で選ぶため、「2×3×5」と「3×2×5」のような順番違いの重複が原理的に発生しません。
- **通常モード**：1〜9 の**異なる3個**（a < b < c）を選び、残りの加算用（d）・減算用（m）は掛け算で使った3個とは**別の数字**（1〜9・0は使わない）だけを選びます。結果は (1つ目, 2つ目, 3つ目, 最後の数) の昇順で並びます。
- **積のみモード**：3個の積をそのまま検索します（d・m は使いません）。通常モードを d=0・m=0 として流用するのではなく、独立した処理で `a × b × c = 入力値` を直接探します。結果は a < b < c の昇順で1段だけ表示します。
- **2数＋／−モード**：掛け算用に 1〜9 の**異なる2個**（a < b）だけを選び（3つ目の c・l は使いません）、加算用（d）・減算用（m）は a・b と異なる別の数字（1〜9・0は使わない）を選びます。`a × b + d` と `j × k − m` を独立した処理で直接探し、(1つ目, 2つ目, 最後の数) の昇順で2段に表示します。

検索範囲が小さい（最大でも数百通り）ため、計算は一瞬で終わります。
